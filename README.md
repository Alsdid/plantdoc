# PlantDoc - 작물 AI 진단 서비스 

YOLOv8 Object Detection 기술을 활용하여 6대 작물(딸기, 토마토, 고추, 파프리카, 오이, 포도)의 병해 및 다량 원소 결핍(질소, 인산, 칼륨 등)을 실시간으로 진단하고 맞춤형 가이드를 제공하는 Streamlit 기반 웹 애플리케이션입니다.

---

## 1. 프로젝트 소개 및 실행 화면

본 프로젝트는 AI 기술을 활용하여 농가의 생산성을 저해하는 작물 질병 및 영양 결핍 상태를 신속하고 정확하게 진단합니다. 직관적인 Web UI를 통해 사용자가 이미지를 업로드하면 실시간으로 객체를 탐지하고 진단 결과와 조치 방법을 안내합니다.

- Language: Python 3.10+

본 서비스는 YOLOv8 모델 가중치 파일(`.pt`)을 사용합니다. 파일 용량 관계로 깃허브 대신 구글 드라이브에 저장되어 있으며, **앱 실행 시 최초 1회 자동으로 다운로드**되도록 설계되어 있습니다. 

수동으로 다운로드가 필요하신 경우 아래 링크를 이용해 주세요.
* https://drive.google.com/drive/folders/1j7nkLdecAq3gRFgfebwyXnF9-MR9gzx8?usp=drive_link
* 다운로드 받은 파일은 프로젝트 내 `weights/` 폴더에 위치해야 합니다.

```bash
# 1. 원격 저장소 복제 (로컬 환경 다운로드)
# 유저님의 실제 깃허브 레포지토리 주소로 바꿔 적으시면 됩니다.

# 2. 필수 의존성 패키지 일괄 설치

# 3. Web 서비스 가동 (실행 시 모델 자동 파싱 가동)
streamlit run app.py

```

## 4. 데이터 파이프라인

본 프로젝트는 6대 작물의 방대한 데이터를 효율적으로 처리하기 위해 팀원 간 3개 작물씩 전담하는 병렬형 데이터 파이프라인을 구축하여 자원 최적화 및 개발 속도를 극대화했습니다.

1. **데이터 소스 및 병렬 수집**: AI 허브의 식물 병 유발 통합 데이터셋을 활용, 각 담당 작물의 고품질 이미지 및 정답지 메타데이터 가공.

2. **클래스 표준화 및 통합**: 파이프라인 최종 단계에서 각자 가공한 6대 작물의 질병 메타 코드(`00`, `a1`, `b6` 등) 모델 인덱스를 단일 웹 서비스 아키텍처에서 추론할 수 있도록 통합 데이터 파이프라인 구조 선언.

3. **데이터셋 분할 검증**: 모델의 일반화 성능 검증 및 오버피팅 차단을 위해 Train, Validation, Test 세트를 **8:1:1** 스케일로 무작위 분할 배정하여 최종 교차 검증 진행.
