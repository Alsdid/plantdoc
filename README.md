# PlantDoc - 6대 작물 병해 및 결핍 AI 진단 서비스 (PBL 최종)

YOLOv8 Object Detection 기술을 활용하여 6대 작물(딸기, 토마토, 고추, 파프리카, 오이, 포도)의 병해 및 다량 원소 결핍(질소, 인산, 칼륨 등)을 실시간으로 진단하고 맞춤형 가이드를 제공하는 Streamlit 기반 웹 애플리케이션입니다.

---

## 1. 프로젝트 소개 및 실행 화면

본 프로젝트는 AI 기술을 활용하여 농가의 생산성을 저해하는 작물 질병 및 영양 결핍 상태를 신속하고 정확하게 진단합니다. 직관적인 Web UI를 통해 사용자가 이미지를 업로드하면 실시간으로 객체를 탐지하고 진단 결과와 조치법을 안내합니다.

## 2. 개발 환경 및 의존성 (Dependencies)

새로운 환경에서도 버전 충돌 없이 구동 가능하도록 최적의 패키지 스펙을 구성하였습니다.
- Language: Python 3.10+
- Core Frameworks: Streamlit (Web UI), Ultralytics (YOLOv8 Engine)
- Image Processing: Pillow (PIL), OpenCV (opencv-python-headless)

## 4. 데이터 파이프라인

본 프로젝트는 6대 작물의 방대한 데이터를 효율적으로 처리하기 위해 팀원 간 3개 작물씩 전담하는 병렬형 데이터 파이프라인을 구축하여 자원 최적화 및 개발 속도를 극대화했습니다.

1. **데이터 소스 및 병렬 수집**: AI 허브의 식물 병 유발 통합 데이터셋을 활용, 각 담당 작물의 고품질 이미지 및 정답지 메타데이터 가공.
2. **클래스 표준화 및 통합**: 파이프라인 최종 단계에서 각자 가공한 6대 작물의 질병 메타 코드(`00`, `a1`, `b6` 등) 모델 인덱스를 단일 웹 서비스 아키텍처에서 추론할 수 있도록 통합 데이터 파이프라인 구조 선언.
3. **데이터셋 분할 검증**: 모델의 일반화 성능 검증 및 오버피팅 차단을 위해 Train, Validation, Test 세트를 **8:1:1** 스케일로 무작위 분할 배정하여 최종 교차 검증 진행.

## 5. 팀원별 역할 분담 

### 팀원 A 
- **데이터 파이프라인 및 모델링**: 데이터셋 분석, 라벨링 좌표 정규화 전처리 가동 및 객체 탐지 알고리즘 설계.
- **AI 학습 인프라 전담**: YOLOv8 모델 아키텍처 커스텀 훈련, 하이퍼파라미터 튜닝 및 Epoch별 평가지표(mAP50-95) 최적화 가동.
- **웹 어플리케이션 설계**: Streamlit 프레임워크 기반 레이아웃 초기 컴포넌트 마이그레이션 및 핵심 구조 프레임워크 기초 빌드.

### 팀원 B
- **데이터 파이프라인 및 모델링**: 데이터셋 분석, 라벨링 좌표 정규화 전처리 가동 및 객체 탐지 알고리즘 설계.
- **AI 학습 인프라 전담**: YOLOv8 모델 아키텍처 커스텀 훈련, 하이퍼파라미터 튜닝 및 Epoch별 평가지표(mAP50-95) 최적화 가동.
- **웹 서비스 프론트엔드 UX 고도화**: Streamlit 어플리케이션 내 사용자 경험 중심 다중 작물 선택 프론트 구조 설계, AI 리포트 보관함 및 세션 스테이트 기반 실시간 대화형 커뮤니티 채팅 인터페이스
